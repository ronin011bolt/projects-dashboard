# Storage Decision Matrix

Use this matrix to decide where information belongs.

## Remember globally

Store in long-term memory only if all are true:
- durable over time
- useful across sessions
- unlikely to create confidentiality risk
- not tied to one project only

Examples:
- communication preferences
- recurring workflow rules
- stable boundaries

## Remember locally

Store in project-local notes/dashboard if any are true:
- specific to one project
- investigative or operational detail
- sensitive enough to avoid broad reuse
- likely to confuse future tasks if generalized

Examples:
- project milestones
- source clusters
- case-specific decisions
- research leads

## Log today only

Store in daily memory if it matters for near-term continuity but not long-term memory.

Examples:
- what was built today
- a deployment completed
- a short-lived blocker

## Do not store

Avoid persistence for:
- passwords
- tokens
- sensitive personal data
- unnecessary identifiers
- highly sensitive unpublished claims

## Rewriting rule

If something feels useful but too sensitive, rewrite it at a higher level before storing.
